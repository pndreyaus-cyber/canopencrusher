# Стурктура CANopenNode и основные объекты

## Объекты
- **OD_t**
- **OD_RAM_t**
- **OD_entry_t** — хранит информацию о записи в объектном словаре. OD_t хранит массив OD_entry_t

- **CO_CANmodule_t** — Какая-то низкоуровневая структура для непосредственного общения с CAN. Инициализируется с помощью функции **CO_CANmodule_init()**
	Основные поля:
	- **CANptr**
	- **CO_CANrx_t\* rxArray** — конфигурационный объект для входящих сообщений (в нем есть callback!)
	- **CO_CANtx_t\* txArray** — конфигурационный объект для исходящих сообщений
	- 
- **CO_t** — коллеция всех CANopenNode объектов. Он содержит ссылки на CO_CANmodule_t и на объекты по типу CO_NMT_t
